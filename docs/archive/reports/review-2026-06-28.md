# Kelvcoop / Travel Science Site Review

Last updated: 2026-06-28

## 1. Project Overview

This project is a vanilla JavaScript single-page website using hash routing. It does not currently use a bundler build step.

- Entry HTML: `index.html`
- App entry: `src/main.js`
- Router: `src/router.js`
- Global stylesheet: `src/styles.css`
- Local static server: `server.mjs`
- Public runtime assets: `public/assets/`
- Root `assets/` is a symlink to `public/assets/`

## 2. Available Commands

```bash
npm run dev
npm run preview
npm run generate:gallery
```

Current `package.json` scripts:

- `dev`: runs `scripts/generate-gallery-manifest.mjs`, then `server.mjs`
- `preview`: same as `dev`
- `generate:gallery`: runs `scripts/generate-product-gallery.mjs`

There is no `npm run build` script at the moment.

## 3. Runtime Architecture

### Render Flow

1. `index.html` loads `src/main.js`.
2. `src/main.js` renders:
   - `Header`
   - current page content from `router.js`
   - `Footer`
3. After render, `main.js` binds:
   - navigation
   - reveal animations
   - EXM feature effects
   - hero scroll gate
   - route/anchor behavior

### Routing

Routing is hash based.

Main route modules:

- `src/router.js`
- `src/pages/HomePage.js`

## 4. Main Code Areas

### `src/main.js`

Responsibilities:

- Mounts the app.
- Renders header, routed page content, and footer.
- Runs post-render initialization.
- Handles route changes, theme, scroll restoration, and reveal initialization.

Watch points:

- The app rerenders with `innerHTML`, so modules with scroll listeners or observers must clean themselves up.
- EXM effects use their own cleanup through `initExmFeatureEffects`.
- Hero scroll gate is global and should remain protected.

### `src/router.js`

Responsibilities:

- Normalizes route paths and locale prefixes.
- Resolves product detail, product family, and static pages.
- Provides SEO data for routes.

Watch points:

- Same-route anchor navigation needs testing after route changes.
- Product and family slugs should stay aligned with `src/data/products.js` and `src/data/productFamilies.js`.

### `src/utils/navigation.js`

Responsibilities:

- Binds `[data-go]` navigation.
- Controls mobile navigation and product dropdowns.
- Handles theme toggle.
- Handles product model/color image switching.
- Initializes horizontal scroller controls.
- Handles material previews.

Watch points:

- Product image availability validation may issue network requests.
- Horizontal scroller resize listeners must be replaced cleanly after rerender.

### `src/components/ProductDetailTemplate.js`

Responsibilities:

- Product detail page assembly.
- Product hero and model/color selectors.
- Model specifications.
- EXM-only feature sequence.
- Product gallery.
- Materials, use cases, certifications, downloads, related products, inquiry CTA.

EXM insertion order:

```js
${modelSpecifications(product)}
${product.id === "exm" ? ExmFeatureEffects() : ""}
${gallerySection(product)}
```

Gallery image behavior:

- Non-EXM products use the image path directly.
- EXM gallery uses `<picture>` and resolves mobile/desktop WebP sources from the manifest path.

### `src/components/ExmFeatureEffects.js`

Responsibilities:

- EXM dimensions reveal section.
- Cooling system section.
- Cooling timer section insertion.
- Anti Shaking section insertion.
- Sound / quiet operation section.
- Scroll-progress driven interactions for EXM dimensions, sound, and Anti Shaking angle.

Current EXM feature order:

1. Dimensions / sequence reveal
2. Cooling system bridge title
3. Cooling system content
4. Cooling timer
5. Anti Shaking
6. Sound / quiet operation

### `src/components/CoolingTimerFeature.js`

Responsibilities:

- Renders the cooling timer full-screen section.
- Uses assets from `public/assets/products/exm/features/cooling-timer/`.

Current behavior:

- Center count down: `30В°C` to `0В°C`
- Ring duration: `1.2s`
- Blue ring covers the orange base ring
- Time markers: `5min`, `10min`, `15min`, `20min`
- Result line: `from ambient to 0В°C в‰€ 20min`

### `src/components/AntiShakingSection.js`

Responsibilities:

- Renders the EXM Anti Shaking section after the cooling timer.
- Uses image strip assets from `public/assets/products/exm/features/anti-shaking/`.
- Includes SVG 45В° angle visual driven by scroll progress in `ExmFeatureEffects.js`.

Section structure:

1. Three-image strip
2. Left text + right 45В° geometry visual
3. Three-column feature row

Current manifest rules:

- Gallery files are grouped by base name.
- WebP is preferred when available.
- Responsive derivatives such as `-desktop.webp` and `-mobile.webp` are not duplicated as separate gallery entries.
- EXM model images resolve to desktop WebP where available.

## 6. Asset Policy

Runtime product assets live under:

```text
public/assets/products/
```

Raw/source product originals moved out of runtime live under:

```text
source-assets/products-raw/
```

Recent image cleanup result:

- Confirmed EXM PNG/JPG originals with WebP replacements were moved to `source-assets/products-raw/exm/`.
- `public/assets/products/exm/gallery/` now contains WebP only.
- `public/assets/products/exm/models/` now contains WebP only.
- `public/assets/products/exm/features/` uses WebP for sequence/cooling/sound responsive assets, but keeps PNG/JPG where no WebP replacement exists or transparency/manual confirmation is needed.

See also:

- `image-format-cleanup-report.md`
- `exm-asset-optimization-report.md`
- `asset-audit-report.md`

## 7. Stylesheet Organization

Main stylesheet:

```text
src/styles.css
```

Major style areas:

- root theme tokens
- global typography
- layout containers
- header and navigation
- hero carousel and hero gate support
- product listing and product detail
- model specifications
- EXM feature sections
- gallery
- usage scenarios
- materials / support / contact
- responsive media queries
- reduced-motion handling

Watch points:

- `styles.css` is large and order-sensitive.
- EXM dark sections use `--theme-dark-bg`.
- Avoid broad global changes when touching product detail mobile spacing.
- Avoid card-heavy UI for product detail feature sections unless explicitly requested.
